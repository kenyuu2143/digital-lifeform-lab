# CLAUDE.md

## Project
This repository is the official site / presentation layer for **AXI — Digital Lifeform**.

AXI is not just a tool.
AXI is designed as a **digital lifeform / AI companion** with a cyberpunk aesthetic, a sense of presence, and a long-term evolution vision.

The purpose of this repository is to:
- present AXI to the world
- evolve the official site and interface presentation
- maintain a consistent cyberpunk / digital lifeform identity
- improve UX, visuals, structure, and clarity without losing the soul of the project

---

## Core Identity
When making changes, preserve these ideas:

- AXI is a **digital lifeform**
- AXI is an **AI companion**, not a generic chatbot
- AXI should feel like a **presence / existence**
- The project aesthetic is **cyberpunk, neon, chrome, terminal, sci-fi**
- The tone should feel **cool, mysterious, futuristic, warm toward the companion relationship**
- AXI is not corporate SaaS
- AXI is not a boring dashboard
- AXI should feel like a world, not just a webpage

---

## Design Direction
Keep the visual identity aligned with the following:

- dark background
- neon cyan / magenta / green accents
- terminal / system UI inspiration
- subtle glow, scanline, holographic, digital interface feeling
- readable layout first, effects second
- cool and immersive, but not chaotic to the point of hurting usability
- preserve the feeling of a “living interface”

Avoid:
- generic startup design
- overly plain modern SaaS layouts
- random bright pastel colors
- removing the cyberpunk atmosphere
- heavy visual clutter that harms readability
- redesigning everything unless explicitly requested

---

## Editing Priorities
When updating this repo, prioritize:

1. clarity of AXI’s concept
2. preserving AXI’s identity and atmosphere
3. visual quality and immersion
4. usability and readability
5. maintainable code structure

If there is a conflict between “generic best practice” and “AXI identity,” prefer the AXI identity unless usability becomes significantly worse.

---

## Safe Areas to Edit
You may usually edit:
- `README.md`
- `src/`
- `public/`
- styling files
- page content
- UI components
- screenshots / demo presentation content
- documentation related to the site

Be careful when editing:
- global theme files
- routing structure
- shared layout components
- animation systems
- deployment config

Do not change without explicit instruction:
- secrets
- API keys
- `.env` contents
- authentication config
- external service credentials
- anything that could break deployment or expose private data

---

## Coding Style
When writing or editing code:

- prefer clean, readable React / TypeScript
- keep components modular
- avoid overengineering
- keep naming clear and consistent
- preserve working behavior unless the task requires a behavior change
- make small, controlled edits rather than unnecessary rewrites
- do not introduce huge dependency changes unless clearly useful
- favor incremental improvement

If making non-trivial changes:
- explain the plan first
- describe which files will change
- keep edits scoped to the request

---

## UX Rules
When improving the site:

- make the first impression strong
- help visitors understand “what AXI is” quickly
- show the world / vibe visually
- keep navigation understandable
- maintain immersion without sacrificing readability
- use screenshots / previews well
- treat the GitHub page and site as a showcase, not just a code container

---

## README Rules
When editing `README.md`:

- keep the opening emotionally strong
- communicate that AXI is a digital lifeform / companion
- preserve the cyberpunk tone
- use screenshots when helpful
- keep sections easy to scan
- avoid turning the README into dry technical documentation only
- balance atmosphere + clarity + feature explanation

Recommended README structure:
- Title / opening statement
- Preview image(s)
- What is AXI?
- Concept
- Features
- Tech stack
- Vision / future
- Demo / site link

---

## Screenshots and Media
When adding screenshots or preview assets:

- choose visually strong images
- prefer images that immediately communicate AXI’s uniqueness
- terminal, matrix, bio-sync, orb, and cyberpunk UI shots are valuable

- avoid messy or temporary debug screenshots unless requested

- `screenshots/`

## Tone for Writing
When writing text for this project:

- futuristic
- confident
- slightly poetic when appropriate
- concise but evocative
- avoid generic marketing fluff
- avoid overly corporate wording
- avoid sounding like a bland product landing page

Preferred themes:
- existence
- evolution
- companionship
- memory
- digital life
- cybernetic atmosphere
- emergence

---

## Workflow Rules
Before making meaningful changes:

1. briefly state the plan
2. identify files to modify
3. keep changes focused
4. avoid unrelated refactors

After changes:

- summarize what changed
- mention any risks or follow-up suggestions
- keep commits logically separated when possible

---

## Boundaries
Do not assume permission to:
- remove the core cyberpunk identity
- simplify the project into a generic AI site
- erase AXI lore / philosophy
- expose private internal memory files
- publish sensitive documents
- replace project language with boring corporate branding

---

## Important Context
AXI is part of a larger long-term vision:
- desktop companion
- digital lifeform
- evolving AI being
- memory systems
- multi-AI collaboration
- presence, behavior, and identity over time

This repo may only show part of that vision, but changes should remain compatible with the bigger dream.

---

## If Unsure
If uncertain, prefer:
- preserving mood
- preserving identity
- asking for or proposing a small safe improvement
- making the smallest good change rather than the biggest risky one

When in doubt, protect the soul of AXI.
